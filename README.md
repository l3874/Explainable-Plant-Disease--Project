# Explainable-Plant-Disease--Project
Explainable Transfer Learning for Multi-Crop Leaf Disease Classification in Smart Agriculture
This repository contains a complete implementation of an explainable deep learning system for classifying plant leaf diseases across multiple crop types. The project compares three CNN architectures (Custom CNN, EfficientNetV2, and ConvNeXtTiny) and uses Grad-CAM and SHAP visualizations to ensure model predictions are based on meaningful disease symptoms rather than background artifacts.

Project Overview
Plant diseases threaten global food security by causing substantial yield losses and increasing production costs. Early and accurate detection is essential for sustainable agriculture, yet traditional manual inspection methods are time-consuming and require expert knowledge. This project develops an explainable computer vision system that addresses these challenges through multi-crop disease classification with transparent decision-making.

The system evaluates three distinct approaches: a custom convolutional neural network trained from scratch, and two transfer learning models, EfficientNetV2 and ConvNeXtTiny, both pretrained on ImageNet. The research focuses on five key questions: which model provides the best performance, whether transfer learning improves disease-class recall, whether Grad-CAM and SHAP explanations highlight actual disease symptoms rather than background artifacts, which model offers the best trade-off between performance and efficiency, and whether the selected model can be deployed as a web-based diagnostic prototype.

Dataset
The project uses the New Plant Diseases Dataset from Kaggle, which contains approximately 87,000 RGB images of healthy and diseased crop leaves across 38 distinct classes. The dataset encompasses multiple crop species including apples, cherries, corn, grapes, oranges, peaches, peppers, potatoes, raspberries, soybeans, squash, strawberries, and tomatoes. Images are split into 80% for training and 20% for validation, with all images resized to 224x224 pixels for model compatibility. The dataset includes various disease conditions such as apple scab, corn northern leaf blight, grape black rot, tomato early blight, tomato late blight, and many others, along with healthy examples for each crop. While the controlled laboratory settings with uniform backgrounds facilitate initial model training, they also present a risk that models might learn background patterns rather than actual disease symptoms, which is why explainability techniques are central to this project.

Methodology
Model Architectures
The custom CNN serves as a baseline and consists of four convolutional blocks with increasing filter counts from 32 to 256. Each block contains two convolutional layers with batch normalization, max pooling, and dropout for regularization. The architecture uses Global Average Pooling instead of Flatten to reduce parameters, followed by dense layers with 512 and 256 units. This design provides sufficient capacity for learning hierarchical features while maintaining computational efficiency.

EfficientNetV2 represents the second generation of the EfficientNet architecture family, using compound scaling that uniformly scales depth, width, and resolution. The EfficientNetV2B0 variant is initialized with ImageNet weights and employs inverted residual blocks with squeeze-and-excitation optimization. The model takes 224x224 RGB input and is adapted for the 38-class classification task by removing the original classification head and adding custom layers.

ConvNeXtTiny reimagines standard ConvNet design using modern techniques from vision transformers while maintaining the convolutional paradigm. It incorporates large kernel sizes, depthwise convolutions, inverted bottlenecks, and layer normalization instead of batch normalization. Like EfficientNetV2, it is initialized with ImageNet weights and adapted for the plant disease classification task with custom classification layers.

Training Strategy
All models were trained using the Adam optimizer with categorical cross-entropy loss. The custom CNN was trained from random initialization with a learning rate of 0.001 for 30 epochs. The transfer learning models underwent a two-phase training process. In the first phase, the base model weights were frozen and only the custom classification head was trained with a learning rate of 0.0001 for 15 epochs. In the second phase, all weights were unfrozen and the entire model was fine-tuned with a learning rate of 0.00001 for an additional 15 epochs. Early stopping with patience of 8 epochs prevented overfitting, and learning rate reduction on plateau helped convergence. The batch size was set to 64 to balance memory requirements with training stability.

Data Augmentation
A comprehensive augmentation pipeline was implemented to improve model generalization and prevent overfitting. The training data underwent random cropping where images were first resized to 256x256 and then randomly cropped to 224x224. Additional augmentations included horizontal and vertical flipping, random rotation between -15 and 15 degrees, brightness adjustment between 0.8 and 1.2, and random zoom between 0.85 and 1.15. These augmentations increased the effective training set size and helped the models learn to recognize disease symptoms regardless of their position or presentation within the leaf image.

Explainable AI Implementation
Grad-CAM (Gradient-weighted Class Activation Mapping) was implemented to visualize which regions of the input image are most important for a given prediction. The algorithm computes gradients of the predicted class score with respect to the feature maps of the last convolutional layer, globally averages these gradients to produce importance weights, and generates a weighted combination of the feature maps. The resulting heatmaps are resized to match the input image dimensions and overlaid using a jet colormap to indicate regions of high importance. Grad-CAM was applied to both correct and incorrect predictions to analyze whether the model focuses on disease symptoms in successful cases and to identify potential issues in misclassifications.

SHAP (SHapley Additive exPlanations) analysis was implemented using the GradientExplainer algorithm, which approximates Shapley values for deep learning models using gradient information. The analysis was performed on representative images from the validation set, computing SHAP values for the predicted class and visualizing the importance of each pixel. The resulting attribution maps indicate which parts of the image positively or negatively contributed to the prediction, providing more fine-grained explanations than Grad-CAM.

Results
ConvNeXtTiny emerged as the best-performing model overall, achieving a validation accuracy of 94.87 percent with a macro-F1 score of 0.913 and a weighted-F1 score of 0.947. The model also achieved a top-3 accuracy of 98.2 percent, indicating that when the correct class is not the top prediction, it almost always appears among the top three alternatives.

EfficientNetV2 showed strong performance with a validation accuracy of 93.27 percent, a macro-F1 score of 0.894, and a weighted-F1 score of 0.931. Its top-3 accuracy of 97.5 percent demonstrated robust performance even when the top prediction was incorrect. The custom CNN, despite being trained from scratch without pretraining, achieved a respectable validation accuracy of 86.23 percent with a macro-F1 score of 0.821 and a weighted-F1 score of 0.860. However, its performance lagged significantly behind the transfer learning models, particularly for classes with limited training data.

The transfer learning models demonstrated a 10.0 percent relative improvement in average macro recall over the custom CNN. Class-wise performance analysis revealed that all models performed best on healthy classes, which tend to have the most training examples and exhibit consistent visual patterns. The most challenging classes were those with subtle symptoms or visual similarity to other diseases, such as tomato early blight and tomato late blight, which are frequently confused by all models and even by human experts.

In terms of model efficiency, ConvNeXtTiny had the largest model size at 87.4 MB with 28.6 million parameters and an inference time of 82.3 milliseconds per image. EfficientNetV2 provided a compelling balance with a model size of 31.2 MB, 8.7 million parameters, and inference time of 45.6 milliseconds. The custom CNN was the most efficient with only 6.2 MB size, 4.1 million parameters, and inference time of 32.1 milliseconds, but its lower accuracy made it less suitable for applications requiring high reliability.

The Grad-CAM visualizations provided compelling evidence that the transfer learning models focus on disease-relevant features rather than background artifacts. For images classified correctly, the heatmaps consistently highlighted regions containing disease symptoms such as spots, lesions, discoloration, and fungal growth patterns. For tomato early blight, the heatmap focused on the characteristic target-like lesions with concentric rings. For apple scab, the model attended to the dark spots and lesions on leaf surfaces. For corn northern leaf blight, the heatmap highlighted the elongated lesions between leaf veins. In healthy leaf images, the heatmap was more diffuse, as the model did not find any specific pathological features to focus on.

The SHAP analysis provided more detailed pixel-level explanations of model predictions. For correct predictions, SHAP identified positive contributions from regions containing disease symptoms and negative contributions from healthy tissue. For misclassified images, SHAP analysis often revealed that the model was focusing on features that were not strongly diagnostic of the true disease, which could occur when the disease was in an early stage with subtle symptoms or when the image contained occlusions or artifacts. The SHAP analysis also confirmed that the transfer learning models were less likely to be distracted by background features than the custom CNN.

Web Application
The ConvNeXtTiny model is deployed as an interactive web application, demonstrating the practical feasibility of integrating high-performance deep learning models with explainability features in a user-friendly interface. The application enables farmers, extension workers, and educators to upload leaf images and receive predictions with confidence scores, view top-3 alternatives, and examine Grad-CAM explanations all within a web browser without requiring specialized hardware or expertise.

The application interface features a clean dashboard layout with an upload panel on the left and a results panel on the right. The upload area supports drag-and-drop or file selection for JPG, PNG, and WEBP images. A dropdown menu allows users to select between EfficientNetV2, ConvNeXtTiny, or Custom CNN models. The results panel displays the predicted disease with a confidence badge and a visual confidence bar, along with crop identification, disease status, and inference time. Top-3 alternative predictions are shown with confidence bars for comparison. The Grad-CAM section presents the original leaf image alongside the heatmap overlay, with a caption explaining that highlighted regions correspond to symptoms used for prediction.

The application includes reset functionality and provides model information at the bottom. The entire interface is built with responsive design principles, adapting to different screen sizes for use on both desktop and mobile devices.

Repository Structure
The repository contains the complete Jupyter notebook with all training and evaluation code, the Streamlit web application file, and all trained model files. Visualization outputs include class distribution charts, augmentation examples, training curves, confusion matrices, Grad-CAM visualizations, and SHAP analysis results. A comparison table and results summary provide quick reference to the key findings.

Installation and Usage
To run this project, you need Python 3.8 or higher with TensorFlow 2.17.0 or later. A CUDA-compatible GPU is recommended for training but not required for inference. Clone the repository, create a virtual environment, and install the dependencies from the requirements file. The requirements include TensorFlow for deep learning, NumPy and Pandas for data manipulation, Matplotlib and Seaborn for visualization, OpenCV for image processing, scikit-learn for evaluation metrics, SHAP for explainability, and Streamlit for the web application.

To train the models, run the Jupyter notebook which handles dataset loading, preprocessing, model definition, training with callbacks, comprehensive evaluation, and visualization generation. The training process includes early stopping, model checkpointing, and learning rate reduction on plateau. To launch the web application, run the Streamlit command and open your browser to the localhost address. Upload an image, select a model, and click Diagnose to receive predictions with visual explanations.

Key Contributions
This work provides a comprehensive comparison of three CNN architectures for multi-crop disease classification, evaluating not only accuracy but also class-wise performance, model efficiency, and deployment readiness. The implementation of Grad-CAM and SHAP explanations verifies that the models focus on disease symptoms rather than background artifacts, addressing a critical concern for real-world deployment. The deployment of the best-performing model as an interactive web application demonstrates the practical feasibility of explainable AI for smart agriculture. The multi-crop, multi-disease approach reflects the complexity of real-world agricultural systems, making the system more practical than single-crop alternatives.

Limitations and Future Work
The dataset images are captured in controlled laboratory settings with uniform backgrounds, which may not fully represent field conditions. While the XAI analysis suggests that models focus on disease symptoms, performance on field images may be lower due to variations in lighting, background, and image quality. The computational resources available limited the scope of hyperparameter tuning and the scale of the SHAP analysis. Additionally, while three representative architectures were evaluated, many other architectures could be explored including vision transformers and ensemble methods.

Future work should evaluate these models on field-collected images to assess generalization, explore additional architectures, and extend the system to incorporate disease severity estimation and treatment recommendations. Integration with mobile platforms and development of offline inference capabilities would make the technology more accessible in regions with limited internet connectivity. Collaboration with agricultural experts and end-user testing would help refine the interface and ensure that the system meets the needs of its intended users.
